# Seed data

`seed.sql` is generated by `generate.ts` and holds enough data to render a
realistic-looking site before the first ingestion run completes.

To regenerate:

```bash
pnpm tsx db/seed/generate.ts > db/seed/seed.sql
```

Once the ingest Worker has run for the first time in a fresh environment,
subsequent writes overwrite the seed values. The seed should never be needed
again except when rebuilding a preview environment from scratch.
